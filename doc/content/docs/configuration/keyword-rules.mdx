## 匹配机制回顾

扩展的关键词匹配机制可用一句话概括：

> **视频标题（小写化后）包含任一关键词（小写化后）作为子串 → 归入该收藏夹。**

详细规则：

- **不区分大小写**：`React` 与 `react` 等价；
- **子串匹配**：关键词 `原神` 命中 `原神 5.0 版本前瞻`；
- **首个命中优先**：一个视频命中多个收藏夹时，按收藏夹顺序取第一个；
- **无跨词匹配**：关键词中不含空格时，不会被标题中的空格拆散；含空格的关键词需在标题中连续出现。

## 同义词策略

### 为什么需要同义词

中文表达灵活，同一主题常有多个说法。只配置一个关键词会漏掉大量视频。

### 推荐做法

为每个主题配置 **3~5 个互补的同义词**：

| 主题 | 推荐关键词组合 |
|------|----------------|
| 游戏攻略 | `攻略` + `教程` + `玩法` + `新手` |
| 前端开发 | `React` + `Vue` + `前端` + `JavaScript` + `TypeScript` |
| 美食 | `美食` + `做饭` + `食谱` + `烘焙` + `料理` |
| 健身 | `健身` + `锻炼` + `训练` + `塑形` |

<Callout type="info">
  可以借助 [AI 关键词提取](/docs/guide/ai-classification) 自动生成候选同义词列表，再人工筛选，比纯手工整理效率高 5~10 倍。
</Callout>

## 应对误命中

当前扩展只做 **正向包含匹配**，不支持显式的排除词。当你发现某个关键词会误命中无关视频时，推荐两种处理思路：

1. **改用更具体的关键词**：例如把 `React` 替换为 `React 教程`、`React Hook`、`React 源码` 等更长的组合词，从根上降低歧义；
2. **用可视化管理手动纠正**：对已经被误分类的少量视频，使用 [可视化拖拽](/docs/guide/drag-and-drop) 直接拖回正确的收藏夹。

## 优先级控制

### 场景

一个视频同时命中 **「游戏」** 与 **「二次元」** 两个收藏夹，应该归到哪个？

### 当前机制

扩展按 **关键词管理界面中收藏夹的顺序** 取第一个命中项。因此你可以通过 **拖动排序** 控制优先级：

- 把 **更具体的收藏夹** 排在前面（如「原神」应排在「游戏」前）；
- 把 **兜底收藏夹** 排在后面（如「综合娱乐」）。

### 实战示例

假设你有如下收藏夹与关键词：

| 顺序 | 收藏夹 | 关键词 |
|------|--------|--------|
| 1 | 原神 | `原神`、`Genshin` |
| 2 | 游戏攻略 | `攻略`、`教程` |
| 3 | 游戏综合 | `游戏` |

对于标题 `原神 5.0 攻略合集`：

- 命中 `原神`（收藏夹 1）✅；
- 命中 `攻略`（收藏夹 2）✅；
- 命中 `游戏`（收藏夹 3）✅；
- **归类结果**：按顺序取第一个命中 → **原神** 收藏夹。

## 关键词的常见陷阱

### ❌ 过于宽泛

`视频`、`up 主`、`大家好` 等词几乎命中所有内容，失去分类意义。

### ❌ 歧义词

`苹果` 可能指水果、也可能指电子产品。解决方案：

- 拆成更具体的关键词（`苹果手机` / `Apple Watch` / `种植苹果`）；
- 或者把 `苹果` 与更有辨识度的词组合使用（通过 **同义词覆盖** 而非单词）。

### ❌ 标题中不出现的词

用户名、内部术语、稿件编号等通常不会出现在视频标题里，配置了也不会命中。

## 性能考虑

- **单夹关键词数量**：建议控制在 **50 个以内**，过多会降低分类速度；
- **总关键词数量**：建议控制在 **500 个以内**，满足绝大多数场景；
- 超出上限时扩展仍能工作，但 **AI 提取关键词** 等功能的响应速度会显著下降。

## 规则维护节奏建议

- **首次配置**：投入 30~60 分钟，借助 AI 完成主干规则；
- **日常微调**：每周花 5~10 分钟调整 5~10 个规则；
- **季度整理**：每 2~3 个月做一次规则大盘清理，合并或删除已经失效的规则。
