## 关键字规则的作用

每个 **目标收藏夹** 维护一组关键字。当视频标题命中任一关键字时，该视频会被归入对应收藏夹。

| 使用方 | 作用机制 |
|--------|----------|
| 收藏夹分析（选项页 → 收藏夹数据分析）| 决定各分类的归属与占比 |
| 通过关键字整理（弹窗底部按钮）| 决定视频会被移动到哪个收藏夹 |
| 自动创建关键字 / AI 辅助生成 | 生成的候选会直接回填到对应收藏夹的关键字列表 |

## 界面布局

选项页 → 关键字管理 Tab 由 **左右两栏** 组成：

- **左侧：收藏夹（FavoriteTag）**：列出你的全部收藏夹，点击选中后，右侧会显示该收藏夹的关键字列表；
- **右侧：关键字（Keyword）**：显示当前选中收藏夹的关键字集合，支持新增、编辑、删除。

顶部还有一个操作按钮区（`OperateButton`），用于触发自动创建关键字等批量操作。

## 操作流程

<Steps>
  <Step>
    ### 进入关键字管理
    通过以下任意方式进入：

    - 右键扩展图标 → **「选项」**，切换到 **「关键字管理」** Tab；
    - 在弹窗中点击底部的 **「自动创建关键字」** 按钮（会直接跳到该 Tab）。
  </Step>
  <Step>
    ### 选择目标收藏夹
    在左侧收藏夹列表中点击要维护的收藏夹，右侧关键字列表会切到对应数据。
  </Step>
  <Step>
    ### 新增 / 编辑 / 删除关键字
    在右侧关键字面板中完成增删改。所有改动会写入本地存储，对后续整理与分析立即生效。
  </Step>
  <Step>
    ### （可选）自动创建关键字
    顶部操作按钮区包含三个元素：

    - **模式选择器**：**「本地算法」**（TF-IDF，快速、免费、离线）或 **「AI 智能」**（准确、需要先配置 AI）；
    - **「为所有创建」**：对你的全部收藏夹依次生成关键字；
    - **「为选中创建」**：仅对当前左侧选中的那个收藏夹生成关键字。

    生成结果会直接回填到右侧的关键字列表，供你继续编辑或删除。
  </Step>
</Steps>

## 匹配规则

- **不区分大小写**：`React` 与 `react` 等价；
- **子串匹配**：关键字 `原神` 会命中 `原神 5.0 版本前瞻` 这类标题；
- **任意命中即归类**：视频标题命中任一关键字即被归入该收藏夹；
- **命中优先级**：当视频命中多个收藏夹时，扩展按收藏夹顺序取第一个命中项；调节优先级详见 [关键字规则进阶](/docs/configuration/keyword-rules)。

## 关键字编写建议

### ✅ 推荐

- **使用具体且有辨识度的词**：如 `原神` `Rust` `烘焙`，命中率高且不易混淆；
- **同一主题配置多个同义词**：如「游戏攻略」类可同时配置 `攻略` `教程` `玩法`；
- **中英文都加一份**：如 `React` 与 `前端框架` 并存可提高命中率。

### ⚠️ 慎用

- **过于宽泛的词**：如 `教程` `分享`，容易误命中；
- **标题中极少出现的词**：如某个 UP 主的真名，建议改用昵称；
- **含标点 / Emoji 的词**：匹配基于纯文本，特殊字符可能降低命中率。

## 本地算法与 AI 的取舍

关键字管理顶部的 **模式选择器** 决定 **「为所有创建 / 为选中创建」** 使用哪种底层算法：

- **本地算法（TF-IDF，默认）**：
  - 在浏览器内完成，无需任何外部请求；
  - 内置中文分词、停用词表（如「视频」「教程」「合集」等会被自动过滤）；
  - 当单个收藏夹视频数 ≤ 2 时，会退化为纯词频算法；
  - 适合 **零成本快速生成候选词** 的日常使用。
- **AI 智能**：
  - 依赖 **自定义配置** 或 **免费额度** 模式已就绪（见 [API Key 配置](/docs/configuration/api-key)）；
  - 质量通常优于本地算法，尤其对 **题材复杂或标题信息量少** 的收藏夹；
  - 会消耗 Token，大规模调用建议先评估成本。

推荐的最佳实践：

1. **首次使用**：选 **「本地算法」** + **「为所有创建」**，快速得到初版关键字；
2. **人工筛选**：删除过于宽泛或歧义的词；
3. **补充质量**：对结果不满意的收藏夹，切到 **「AI 智能」** 再单独 **「为选中创建」** 一次；
4. **长期维护**：每整理一轮后，回来微调 5~10 分钟。
