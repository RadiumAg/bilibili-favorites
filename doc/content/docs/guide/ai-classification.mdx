**AI 智能分类** 是扩展在弹窗/侧边栏底部提供的 **「🤖 AI 整理」** 按钮，利用大模型根据视频标题与收藏夹名称自动归类。

## 工作原理

<Callout type="info">
  AI 功能 **仅发送视频标题与目标收藏夹名称**，**不会发送** UP 主信息、观看记录或任何账号数据。更完整的隐私说明见 **隐私与安全** 章节。
</Callout>

- **请求内容**：默认收藏夹下所有视频的标题 + 你所有收藏夹的名称 + 预设 Prompt；
- **接口协议**：OpenAI Chat Completion 兼容（走流式读取）；
- **调用时机**：仅在你 **主动点击「🤖 AI 整理」并在 5 秒内二次确认** 后才触发，不会后台自动调用；
- **兜底策略**：若 AI 返回的目标收藏夹名称不在你的收藏夹列表中，该视频会被兜底移入 **默认收藏夹**，并在 Toast 详情中列出，方便你人工复核。

## 前置条件

在使用之前，需要完成 **以下二选一**：

- **自定义配置**：在选项页 → 配置 → 自定义配置中填入 **模型名 + API Key**（Base URL 会根据所选服务商自动填充）；
- **免费额度**：在选项页 → 配置 → 切换到 **「免费额度」**，扩展将使用内置 AIGate 共享配额。

未配置时点击「🤖 AI 整理」会弹出以下提示，并自动打开配置页：

> 请先在设置页面配置 AI 或切换到免费额度

完整配置说明见 [API Key 配置](/docs/configuration/api-key) 与 [模型选择与推荐](/docs/configuration/model-selection)。

## 使用流程

<Steps>
  <Step>
    ### 确认默认收藏夹已设置
    扩展以默认收藏夹为整理起点；若尚未设置默认收藏夹，会弹出提示。
  </Step>
  <Step>
    ### 点击「🤖 AI 整理」
    入口在弹窗/侧边栏底部；按钮旁 `?` 展开更多说明。
  </Step>
  <Step>
    ### 确认 Token 消耗（首次点击）
    首次点击仅弹出 Toast：

    > ⚠️ Token 消耗提醒：此操作会将所有视频标题发送给 AI 进行分析，可能消耗大量 Token。再次点击确认执行。

    **5 秒内再次点击** 同一按钮才会真正发起请求；超时则需重新进入确认态。
  </Step>
  <Step>
    ### 等待 AI 分析
    遮罩层会显示 loading 动画，提示文案为 **「AI 分析中...」**；扩展会：

    1. 自动分页拉取默认收藏夹的全部视频；
    2. 将标题与收藏夹名批量发送给 AI；
    3. 通过流式接口逐步拼接 AI 返回的 JSON；
    4. 解析后进入移动阶段。
  </Step>
  <Step>
    ### 等待移动执行
    移动阶段文案变为 **「移动中...」**，每条移动间隔 100ms 避免接口限流；执行期间可以点击遮罩中的 **「取消」** 中断。
  </Step>
  <Step>
    ### 查看结果面板
    完成后展示每条视频的归类结果（视频标题 → 目标收藏夹）；同时 Toast 汇总：

    - **成功**：已被正常移动的视频数；
    - **失败**：移动接口返回异常的视频数；
    - **兜底**：AI 返回的目标收藏夹名称不存在，已改归默认收藏夹的视频数（Toast 详情中会逐条列出）。
  </Step>
</Steps>

## AI 提取关键字

除了「AI 整理」外，扩展还在选项页 → **关键字管理** Tab 中提供 **AI 辅助生成关键字** 能力（入口由 `OperateButton` 承载）。你可以基于当前选中收藏夹的视频标题批量生成候选关键字，然后在界面中编辑保留。详细用法见 [关键字管理](/docs/guide/keyword-management)。

## 取消与错误处理

- **取消操作**：遮罩中的 **「取消」** 按钮会：
  - 中断 AI 流式读取；
  - 中断后续的移动接口调用；
  - 已经移动成功的视频不会回滚；
- **AI 返回格式异常**：扩展期望响应包含 `[...]` JSON 数组；若无法解析，会 Toast 报错 **「AI 返回的数据格式错误，请重试」** 并附带原始返回内容；
- **用户取消**：会 Toast 「已取消」提示；
- **接口报错**：Toast 标题为「整理失败」，描述为错误信息，详情区展示更深的错误栈。

更多详细排障流程见 **故障排除** 章节。

## 性能与成本

- **Token 消耗量** 与默认收藏夹中视频数量成正比：每条标题平均约 20~50 tokens，完整 Prompt 还会包含收藏夹名称列表；
- **大批量整理**：若默认收藏夹视频数超过 500 条，建议：
  - 先用「通过关键字整理」处理能命中的大部分；
  - 剩余未命中的再用 AI 整理，以节省成本；
- **服务商限流**：少数服务商对流式响应有 RPM 限制，遇到限流可切换服务商或等待后重试。
