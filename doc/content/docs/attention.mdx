本章节汇总了使用 **B 站收藏夹整理工具** 过程中的常见问题、安全与隐私注意事项，以及标准化的故障排除流程。在提交 Issue 之前，建议先按本章节排查。

## 使用前须知

<Callout type="warn" title="开始使用前，请确认以下事项">
  - 扩展依赖 **B 站页面的登录态**，必须保持至少一个已登录的 `bilibili.com` 标签页；
  - 为避免频繁调用官方接口，收藏数据 **默认缓存 24 小时**，需要最新数据请手动点击「刷新」；
  - 关键字提取的 **本地算法（TF-IDF）** 与 **AI 智能** 是两种并列模式，需在「关键字管理」页顶部的 **模式选择器** 中显式切换；
  - 本扩展基于 **Chrome Extension Manifest V3** 构建，部分旧版 Chromium 浏览器（< 114）可能不兼容。
</Callout>

## 常见问题

以下问题按出现频率排列，可逐项排查。

<Accordions>
  <Accordion title="打开扩展后提示「需要登录」？" id="faq-login">
    扩展通过 B 站页面的 Cookie 识别登录态，而非独立维护登录会话。请按以下步骤处理：

    <Steps>
      <Step>
        ### 打开任意 B 站页面
        在浏览器中新开一个标签页，访问 [bilibili.com](https://www.bilibili.com)。
      </Step>
      <Step>
        ### 完成登录
        若未登录，请先扫码或使用账号密码完成登录。
      </Step>
      <Step>
        ### 刷新 B 站页面
        按下 `F5` 或 `Ctrl/⌘ + R` 刷新该 B 站页面，确保 Cookie 被正确写入。
      </Step>
      <Step>
        ### 重新唤起扩展
        关闭扩展面板后，再次点击图标重新打开。
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="为什么收藏数据看起来不是最新的？" id="faq-cache">
    为降低对 Bilibili 官方接口的压力，扩展默认会对收藏数据进行 **24 小时** 的本地缓存。

    如需强制拉取最新数据：

    - 进入 **收藏夹分析** 页，点击右上角的 **「刷新」** 按钮；
    - 或在扩展配置中手动清空缓存后重新打开扩展。
  </Accordion>

  <Accordion title="AI 分类结果不理想怎么办？" id="faq-ai-quality">
    AI 分类质量主要取决于 **模型能力** 与 **关键词规则**。建议：

    - 优先选用推理能力较强的模型（如 GPT-4o、DeepSeek V3、星火 4.0 Ultra 等）；
    - 在 **「关键词管理」** 中人工微调 AI 生成的候选关键词，删除过于宽泛或歧义的词；
    - 对分类边界模糊的内容，可以通过 **可视化拖拽** 手动调整。
  </Accordion>

  <Accordion title="AI 功能必须付费吗？" id="faq-ai-cost">
    不强制付费。你有两种零成本方案：

    - **使用免费额度模型**：如 **讯飞星火大模型** 提供免费调用额度，足够日常使用；
    - **完全不启用 AI**：在「关键字管理」页的模式选择器中切换到 **「本地算法」**，扩展将使用内置 **TF-IDF 算法** 离线提取关键字。
  </Accordion>

  <Accordion title="批量整理时出现部分视频移动失败？" id="faq-batch-fail">
    常见原因及对应处理：

    - **目标收藏夹已满**：B 站单个收藏夹有容量上限，请先清理或新建收藏夹；
    - **视频已失效**：源收藏夹中的视频被 UP 主删除或下架，将无法移动；
    - **接口限流**：短时间内操作过于频繁，等待 1~2 分钟后重试即可；
    - **登录态失效**：见上文「提示需要登录」的处理流程。

    操作完成后扩展会显示 **成功 / 失败计数**，失败条目可再次执行整理即可。
  </Accordion>

  <Accordion title="侧边栏无法打开或显示空白？" id="faq-sidepanel">
    请依次检查：

    1. 浏览器版本是否满足 Chrome 114+；
    2. 是否通过右键扩展图标 → **「在侧边栏中打开」** 的方式唤起；
    3. 若仍异常，尝试在 `chrome://extensions/` 中 **禁用并重新启用** 扩展。
  </Accordion>

  <Accordion title="支持哪些浏览器？" id="faq-browsers">
    基于 **Chrome Extension Manifest V3** 构建，理论上兼容所有 Chromium 内核浏览器，包括：

    - Google Chrome（114+）
    - Microsoft Edge
    - Arc
    - Brave
    - Vivaldi

    Firefox 暂不支持（Manifest V3 生态差异较大）。
  </Accordion>
</Accordions>

## 安全与隐私最佳实践

<Callout type="info" title="本扩展的隐私承诺">
  - **不收集** 任何个人身份信息；
  - **不上传** 用户数据至任何第三方服务器；
  - 所有数据仅保存在 **浏览器本地**（IndexedDB + Chrome Storage）；
  - 网络请求仅用于访问 **Bilibili 官方 API** 与 **OpenAI 兼容 API**（可选）。
</Callout>

在此基础上，建议你养成以下使用习惯以进一步提升安全性：

### API Key 管理

- **仅在个人设备使用**：API Key 存储于 Chrome Storage，共用设备可能导致 Key 泄露；
- **使用最小权限 Key**：若服务商支持，生成仅具备模型调用权限的专用 Key；
- **定期轮换**：长期使用同一把 Key 时，建议每 3~6 个月手动轮换一次；
- **发现异常立即吊销**：一旦怀疑 Key 泄露，应立即在服务商后台吊销并生成新 Key。

### 账号与登录态

- **不要在不受信任的设备** 登录 B 站后再安装扩展；
- 若设备借给他人使用，建议先 **在扩展配置中清空缓存**，避免收藏数据被意外浏览；
- 离开设备前 **锁屏** 或退出 Chrome 用户档案，避免他人操作收藏夹。

### 数据备份与清理

- 在执行 **大规模批量整理** 之前，建议先使用少量视频进行小范围验证；
- 定期导出 B 站官方收藏数据作为冷备份，避免误操作带来的长期影响；
- 卸载扩展时，Chrome 会自动清理扩展对应的本地数据；如需手动清理，可在扩展详情页的 **「网站数据」** 中操作。

### 权限最小化

扩展仅申请运行所必需的权限（`storage`、`sidePanel`、`bilibili.com` 域名访问等），**不会** 申请：

- 读取其他网站的浏览历史；
- 访问剪贴板或摄像头；
- 修改你的浏览器主页或搜索引擎。

如在安装时发现额外权限请求，极可能是 **仿冒版本**，请立即卸载并从官方 Chrome Web Store 重新安装。

## 故障排除

当扩展出现异常时，推荐按 **从简到繁** 的顺序排查。

### 第一步：基础自检

<Steps>
  <Step>
    ### 确认登录态
    打开任意 B 站页面，确认账号已登录且页面可正常浏览。
  </Step>
  <Step>
    ### 确认网络环境
    确保可正常访问 `bilibili.com`；若启用了 AI 功能，确认可访问所选模型服务商的 API 端点。
  </Step>
  <Step>
    ### 确认扩展版本
    在 `chrome://extensions/` 查看扩展版本号，确保是最新版（低版本可能已知 Bug 未修复）。
  </Step>
</Steps>

### 第二步：清理缓存与状态

若基础自检无异常但问题仍存在：

<Steps>
  <Step>
    ### 清空扩展缓存
    在扩展 **配置页** 找到「清空缓存」或「重置数据」按钮，执行后重新打开扩展。
  </Step>
  <Step>
    ### 重启扩展
    前往 `chrome://extensions/`，关闭本扩展的开关，再重新打开。
  </Step>
  <Step>
    ### 重启浏览器
    彻底关闭 Chrome（包括所有窗口）后重新打开，部分后台异常可被清除。
  </Step>
</Steps>

### 第三步：收集诊断信息

如仍无法解决，请在提交 Issue 前收集以下信息，便于维护者快速定位：

- **浏览器类型与版本**（例如 Chrome 126.0.6478.127）；
- **扩展版本号**（见 `chrome://extensions/`）；
- **操作步骤与预期结果**，以及实际观察到的现象；
- **控制台日志**：
  - 在扩展弹窗 / 侧边栏上右键 → **「检查」**，切到 **Console** 面板，截图或复制红色 Error 日志；
  - 在 B 站页面按 `F12` 打开 DevTools，同样复制 Console 面板的错误日志；
- **截图或录屏**（如涉及 UI 异常）。

<Callout type="warn">
  提交日志或截图前，请 **先脱敏** 处理，例如抹去 API Key、Cookie、uid 等敏感信息。
</Callout>

完成信息收集后，可以前往扩展的 **开源仓库 Issues** 页按模板提交：

- 使用 **清晰的标题** 概括现象（如 `[Bug] 批量整理时部分视频提示「接口限流」`）；
- 在正文中粘贴 **诊断信息**；
- 若问题可稳定复现，请给出 **最小复现步骤**。
